Webhooks

What are webhooks?

Webhooks allow your application to receive real-time notifications when events occur in your HamsukyPay account. When an event happens, we'll send a HTTP POST request to the URL you configure with details about the event.

Common use cases include: Getting notified when a payment succeeds or fails, synchronizing customer data with your systems, triggering fulfillment processes after successful payments, and updating your application's UI in real-time.

Implementation Guide

Follow these steps to implement webhook handling in your application:

1. Create an endpoint on your server

Set up an endpoint on your server that can receive POST requests from HamsukyPay. Here's an example in Node.js with Express:

const express = require('express');
const app = express();
app.use(express.json()); // Parse JSON request body

app.post('/webhook/payment', (req, res) => {
  const signature = req.headers['x-hamsukypay-signature'];
  const event = req.body;
  
  // Verify the signature to ensure the request is from HamsukyPay
  if (!verifySignature(signature, JSON.stringify(req.body), YOUR_WEBHOOK_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  
  // Handle the event based on its type
  switch (event.type) {
    case 'payment.successful':
      // Handle successful payment
      const payment = event.data;
      console.log(`Payment successful: ${payment.reference}`);
      break;
      
    case 'payment.failed':
      // Handle failed payment
      break;
      
    // Handle other event types
  }
  
  // Always return a 200 response to acknowledge receipt of the webhook
  res.status(200).send('Webhook received');
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});

2. Verify webhook signatures

Always verify the signature in the "x-hamsukypay-signature" header to ensure the webhook came from HamsukyPay:

function verifySignature(signature, payload, secret) {
  const crypto = require('crypto');
  const hmac = crypto.createHmac('sha256', secret);
  const expectedSignature = hmac.update(payload).digest('hex');
  
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

3. Implement idempotency

HamsukyPay may send the same webhook multiple times to ensure delivery. Implement idempotency by checking the event ID and ignoring duplicate events.

4. Respond quickly

Your webhook handler should respond with a 2xx status code as quickly as possible. If you need to perform time-consuming tasks, do them asynchronously after sending a response.
